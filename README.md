## ภาพรวม (Overview)

เครื่องมือวิเคราะห์รูปคลื่นความสั่นสะเทือนผ่าน Jupyter Notebook สำหรับติดตามสภาพการทำงานของเครื่องจักรหมุน (Rotating Machinery) Notebook นี้จะประมวลผลข้อมูลความเร่ง (Accelerometer) ดิบรูปแบบ .txt, ตรวจสอบคุณภาพของสัญญาณ, ตัดข้อมูลที่ผิดปกติ (Outliers) ออก, คำนวณค่าตัวชี้วัดความสั่นสะเทือนที่สำคัญ, และสร้างกราฟวิเคราะห์พร้อมกับรายงานรูปแบบ CSV ที่สอดคล้องกับการแบ่งโซนตามมาตรฐาน ISO 10816-3

ออกแบบมาสำหรับการวัดผล 2–5 ระยะเวลา (Periods) ผลวิเคราะห์เหมาะสำหรับการสังเกตแนวโน้ม (Trend) และช่วยตัดสินใจในการซ่อมบำรุง ไม่ได้ออกแบบมาเพื่อสร้างโมเดลทำนายล่วงหน้า (Predictive Modelling) อย่างเต็มรูปแบบ ดังนั้น **ในโปรเจกต์นี้จึงไม่มีการแบ่งชุดข้อมูลเป็น Train set และ Test set**

การทำงานของระบบจะเน้นหนักไปที่การเฝ้าติดตามสภาพเครื่องจักร (Condition Monitoring) แบบ **Rule-based** ภายใต้มาตรฐานสากล โดยมีขั้นตอนหลัก ได้แก่:
1. **การประมวลผลและทำความสะอาดสัญญาณดิบ (Signal Processing & Cleaning):** กรองค่ากระโดดผิดปกติที่เกิดจากสัญญาณรบกวน (Outliers/Spikes) ตรวจสอบความถูกต้องของสัญญาณ (Validation) 
2. **การคำนวณตัวชี้วัดทางสถิติ (Metrics Computation):** เช่น ค่า RMS, Peak, Crest Factor (CF) และการคำนวณหาความถี่จากสัญญาณ (FFT) 
3. **การพิจารณาโซนตามมาตรฐาน (ISO 10816-3):** นำค่าที่ผ่านกระบวนการมาคัดแยกจัดหมวดหมู่โซนมาตรฐาน (A, B, C, D) 
4. **การวิเคราะห์ดูแนวโน้ม (Trend Analysis):** นำจุดข้อมูลในหลายช่วงเวลา (2–5 periods) มาพล็อตหาความลาดชัน (Slope) เพื่อดูพฤติกรรมความเสียหาย และวางแผนซ่อมบำรุงล่วงหน้า

## รูปแบบไฟล์ข้อมูลนำเข้า (Input File Format)

ไฟล์ข้อความธรรมดา (.txt) สำหรับข้อมูลรูปคลื่นที่ส่งออกจากระบบรับข้อมูล โดยแต่ละไฟล์ต้องประกอบด้วย:

- บรรทัดส่วนหัว (Header) ที่มีฟิลด์ `Equipment:`, `Meas. Point:`, `Date/Time:`, และ `Amplitude:`
- ข้อมูลตัวเลข 2 คอลัมน์: เวลา (ms) และ ความเร่ง (G) ในแต่ละบรรทัด

ตัวอย่างส่วนหัวของไฟล์:

```text
Equipment:    Motor Compressor OAH-06_A
Meas. Point:  A_CH-06
Date/Time:    28-Jun-24 08:56:47  Amplitude: G
---
0.0000   0.0123
0.1000  -0.0045
...
```

รูปแบบวันที่ที่รองรับ: `DD-Mon-YY HH:MM:SS`, `DD-Mon-YYYY HH:MM:SS`, `DD-MM-YYYY HH:MM:SS`, `YYYY-MM-DD HH:MM:SS`

## เซสชั่นต่างๆ (Sessions)

Notebook ถูกแบ่งออกเป็นส่วนๆ (Session) ซึ่งต้องรันตามลำดับ

### Session 0 — Setup and Imports
นำเข้าไลบรารีทั้งหมดและตั้งค่าตัวแปรมาตรฐานสำหรับการจัดการ Outlier (ต้องรันก่อน Session อื่นๆ)

### Session 0b — Parser
กำหนดฟังก์ชัน `parse_waveform_txt()` สำหรับอ่านไฟล์ข้อมูล .txt

### Session 0c — Signal Processing
กำหนดฟังก์ชันหลักได้แก่: `remove_outliers()`, `compute_metrics()`, `iso_zone()`, `compute_fft()`, `top_freqs()`, และ `style_ax()`

### Session 1 — Load Files
เปิดหน้าต่างเลือกไฟล์ (ด้วย tkinter) เพื่อเลือกไฟล์ .txt จำนวน 2–5 ไฟล์ โดยไฟล์จะถูกจัดเรียงตามลำดับเวลาอัตโนมัติ

### Session 2 — Outlier Config
ตั้งค่าพารามิเตอร์ 3 ตัว สำหรับการจัดการ Outlier (สามารถแก้ไขได้ก่อนรัน Session 3)

| พารามิเตอร์ | ค่าเริ่มต้น | คำอธิบาย |
|---|---|---|
| `TRIM_START_MS` | 1.0 | ตัดข้อมูลช่วงเริ่มต้น (Startup transient) ก่อนเวลาที่กำหนด (ms) (ตั้งค่าเป็น 0 เพื่อปิดการใช้งาน) |
| `IQR_MULTIPLIER` | 8.0 | ตัวคูณช่วง IQR สำหรับตรวจจับค่าแหลมคลื่น (Spike) ค่าน้อยจะตัดจุดข้อมูลออกมาก |
| `REPLACE_METHOD` | interpolate | วิธีการแทนที่จุดข้อมูลที่ถูกลบไป: `interpolate`, `median`, หรือ `zero` |

### Session 3 — Parse and Clean Data
โหลดไฟล์ข้อมูลแต่ละไฟล์, ทำการลบ Outlier, และแสดงผลกราฟวินิจฉัยข้อมูล (Outlier Diagnostic plot จำนวน 3 กราฟย่อย) ของแต่ละไฟล์ก่อนทำไฟล์ต่อไป ข้อมูลทั้งหมดถูกเก็บไว้ในตัวแปร `all_ds`

กราฟวินิจฉัยข้อมูลประกอบด้วย:
1. ข้อมูลดิบเทียบกับข้อมูลที่คลีนแล้ว แสดงช่วงที่ถูกตัดทิ้งจุดเริ่มต้น (Trim region), จุดที่ถูกตรวจพบว่าเป็น Spike, และเส้นช่วง IQR
2. ฮิสโตแกรมแอมพลิจูดก่อนและหลังการคลีนข้อมูล
3. กราฟแท่งเปรียบเทียบค่า RMS, Peak, Crest Factor และ Kurtosis ระหว่างก่อนและหลังการคลีน

### Session 3b — Raw Data Plots
แสดงกราฟข้อมูลดิบ (4 กราฟย่อย) สำหรับแต่ละไฟล์:
1. รูปคลื่นทั้งหมด พร้อมเส้นอ้างอิงค่า RMS และ Peak
2. รูปคลื่นแบบขยายดูรายละเอียด (ช่วง 200 ms แรก)
3. สเปกตรัมความถี่ FFT (0–2000 Hz) แสดงผล 5 ความถี่สูงสุด
4. ฮิสโตแกรมความถี่แอมพลิจูด (Amplitude histogram) พร้อมตารางค่าทางสถิติ

### Session 3c — Outlier Removal Audit
🛡️ ตรวจสอบรายการข้อมูลที่ถูกแก้ไขและเปรียบเทียบค่าทางสถิติ:
- รายงานรายการ Spike ที่ตรวจพบพร้อมเวลา (Timestamp) และค่า Amplitude
- ตารางสรุปเปรียบเทียบค่า RMS, Peak และ Kurtosis ระหว่างก่อนและหลังการคลีนข้อมูล (Raw vs Cleaned)

### Session 4 — Data Validation
ตรวจสอบคุณภาพไฟล์แต่ละไฟล์อัตโนมัติจากเกณฑ์ 5 ข้อ และคัดกรองการวัดที่ไม่ตรงตามเกณฑ์ ไฟล์ที่ไม่ผ่านเกณฑ์จะถูกคัดออกจากการวิเคราะห์ทั้งหมด นอกเสียจากจะถูกตั้งค่าบังคับ

เกณฑ์การตรวจสอบคุณภาพ:

| ข้อกำหนด | ค่าจำกัด (เริ่มต้น) | สิ่งที่ตรวจพบ |
|---|---|---|
| `PEAK_MAX_G` | 50 G | เซ็นเซอร์รับค่าเกินกำหนด หรือได้รับแรงกระแทก |
| `CF_RAW_MAX` | 30 | ค่า Crest Factor สูงผิดปกติ |
| `KURT_RAW_MAX` | 500 | มีการกระแทกในจุดเดียว (ไม่ใช่การสั่นแบบต่อเนื่อง) |
| `FLAT_RATIO_MAX` | 30% | ข้อมูลขาดหาย (เซ็นเซอร์หลุดหรือสายไฟมีปัญหา) |
| Removed % | > 5% | มีการลบข้อมูล Outlier มากเกินกว่าสัดส่วนความยาวสัญญาณปกติ |

*สามารถปรับเปลี่ยนเกณฑ์ได้ที่ส่วนบนสุดของโค้ด Session 4*

### Session 4b — Dataset Selection

## ส่วนอ้างอิงมาตรฐาน ISO 10816-3 (ISO 10816-3 Zone Reference)

เครื่องมือวิเคราะห์นี้ได้อ้างพิจารณาตัวบ่งชี้จากค่า Crest Factor เพื่อเทียบโซนชี้วัดตาม ISO 10816-3 เพื่อให้สามารถวิเคราะห์ความรุนแรงได้ หากจะประเมินผลตามมาตรฐานโดยตรงจำเป็นต้องมีการวัดค่าความเร็วเชิงเส้น (Velocity mm/s) แบบแยกต่างหาก

| ตำแหน่งโซน (Zone) | ระยะค่า CF | สภาวะของเครื่องจักร |
|---|---|---|
| A | < 2.5 | ใหม่ หรือ ทำงานปกติและสมบูรณ์ดี (Good) |
| B | 2.5 – 3.0 | ทำงานได้อย่างปกติต่อเนื่องในระยะยาว |
| C | 3.0 – 4.0 | ระมัดระวังเป็นพิเศษ และ มีการจำกัดการใช้งาน ให้เฝ้าระวังอย่างใกล้ชิด |
| D | > 4.0 | เสี่ยงต่อความเสียหายจากการพังทะลาย ให้รับมือและหยุดใช้งานทันที |

## ข้อจำกัดที่ควรทราบ (Known Limitations)

- การคำนวณของ ISO 10816-3 ใช้ระดับความเร็ว (mm/s) แต่ไม่ใช่ความเร่ง การคำนวณจากค่า Crest Factor นำมาใช้เป็นแค่การตรวจสอบอย่างคร่าวๆ เท่านั้น
- การคำนวณหาจุดตัดแนวโน้ม (Trend analysis และ slope) ควรมีการทดสอบย้อนหลังอย่างน้อย 10 รอบเพื่อให้ข้อมูลมีความน่าเชื่อถือทางสถิติ กรณีมีแค่ 2 ช่วงเวลาจะแสดงทิศทางแค่เป็นไกด์ไลน์
- อัตราตัวเลขในการคัดกรองข้อมูลจากเกณฑ์ใน Session 4 เป็นสัดส่วนเริ่มต้น ผู้ใช้ควรปรับตั้งค่าให้ตรงตามประเภทอุปกรณ์และระดับการสั่นที่ยอมรับได้
- ความแม่นยำย่านความถี่ของการคำนวณ FFT มีระดับสัมพันธ์ต่อความกว้างและสัดส่วนที่บันทึกสัญญาณ เช่น ที่ความถี่อัปเดต (Fs) = 10,000 Hz, จำนวนข้อมูล (N) = 4096 Samples (ใช้ระยะเวลาทั้งหมด 533 ms), ระดับความคลาดเคลื่อนจะมีสัดส่วนเป็น ~1.9 Hz / Bin
- โค้ดนี้ไม่มีกระบวนการวิเคราะห์แบบ Enveloping (Demodulation) และไม่ครอบคลุมถึงผลความถี่ความเสียหายต่อระบบตลับลูกปืน (Bearing defect frequencies เช่น BPFO/BPFI และ BSF) ผู้ใช้งานจำเป็นจะต้องเตรียมเรขาคณิตลูกปืนและคำนวณแยกต่างหากเพื่อนำมาประเมินร่วมกัน

## แนะนำสิ่งแรกที่สำคัญเมื่อเข้าระยะการประเมินในโซน C หรือโซน D

1. วัดความเร็วการสั่น (Velocity mm/s) เพื่อประเมินค่าเปรียบเทียบมาตรฐาน ISO 10816-3 โดยตรง
2. ทำวินิจฉัยรูปคลื่น (Envelope analysis/Demodulation) เพื่อดูจุดความถี่หรือสัญญาณบกพร่องในตลับลูกปืน
3. นำอัตราการหมุน (Shaft speed) และเรขาคณิตของตลับลูกปืนไปพิจารณาเทียบในความถี่ของ FFT ร่วมด้วย BPFO, BPFI และ BSF
4. ประเมินสภาพสารหล่อลื่น (Lubrication condition) กับอุณหภูมิที่เกิดขึ้นบริเวณตลับลูกปืน
5. ถ้าตรวจสอบพบว่าแนวโน้มในอนาคตอันใกล้ Crest Factor ค่าเกิน 4.0 ควรดำเนินการวางแผนงานเปลี่ยนอะไหล่ตลับลูกปืนทันที (Bearing replacement planning)
