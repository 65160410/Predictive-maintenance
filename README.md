# คู่มือการอ่านไฟล์ `vibration_full_report.csv`

ไฟล์ `vibration_full_report.csv` เป็นไฟล์สรุปผลการวิเคราะห์ข้อมูลความสั่นสะเทือน (Vibration Analysis) ทั้งหมดที่ถูกสร้างขึ้นจากโปรแกรม `main.py` โดยแบ่งข้อมูลออกเป็น 8 ส่วน (Sections) ดังนี้

---

## 1. VIBRATION METRICS (clean)
สรุปค่าทางสถิติและตัวชี้วัดความสั่นสะเทือนของแต่ละรอบเวลา (Period) **หลังจากตัดสัญญาณรบกวน (Spike/Outlier) ออกไปแล้ว**
*   **Period**: ช่วงเวลาที่วัดข้อมูล
*   **Date/Time, Equipment, Meas. Point**: ข้อมูลทั่วไปของเครื่องจักรและจุดที่วัด
*   **N Raw, Startup Trimmed, Spikes Removed, N Clean**: จำนวนจุดข้อมูลดิบ, ข้อมูลช่วงเริ่มต้นที่ตัดทิ้ง, จำนวน Spike ที่ตัดทิ้ง, และจำนวนข้อมูลที่เหลือหลังทำความสะอาด
*   **RMS, Peak, Peak-to-Peak (G)**: ค่าความรุนแรงของการสั่นสะเทือน (Root Mean Square, ค่าสูงสุด, และผลต่างค่าสูงสุด-ต่ำสุด)
*   **CF (Crest Factor)**: อัตราส่วนระหว่าง Peak กับ RMS **(ค่าปกติควร < 3, ถ้า > 4 อาจหมายถึงลูกปืน/Bearing เริ่มมีปัญหา)**
*   **Kurtosis / Skewness**: ความโด่ง/ความเบ้ของสัญญาณ **(ค่า Kurtosis > 3 บ่งบอกว่ามีแรงกระแทกเป็นจังหวะหรือ Impulsive)**
*   **ISO Zone**: โซนความรุนแรงตามมาตรฐาน ISO 10816-3 (A=ดี, B=ปกติ, C=เฝ้าระวัง, D=อันตราย)

## 2. TREND (all periods)
แสดงแนวโน้มของค่าพารามิเตอร์สำคัญเทียบตั้งแต่รอบเวลาแรกสุดจนถึงรอบเวลาล่าสุด
*   **Delta / Delta %**: ค่าที่เปลี่ยนแปลงไป และเปอร์เซ็นต์การเปลี่ยนแปลง
*   **Flag**: การแจ้งเตือนอัตโนมัติ (เช่น 🔴 CF>4 — Bearing damage likely แจ้งเตือนโอกาสลูกปืนเสียหาย)

## 3. TOP DOMINANT FREQUENCIES
ความถี่ (Hz) และขนาด (G) ที่สูงที่สุด 8 อันดับแรกในแต่ละช่วงเวลา จากการทำ FFT (Fast Fourier Transform)
*   **New in last period**: แจ้งเตือน ⚠ New หากพบความถี่แปลกใหม่ที่ไม่เคยปรากฏในอดีต (อาจชี้ถึงความเสียหายใหม่)

## 4. ASSESSMENT SUMMARY
คำแนะนำและการประเมินสถานะของเครื่องจักรในปัจจุบัน (ดูจากข้อมูลล่าสุดเป็นหลัก)
*   **Section A & B**: ข้อมูลเครื่องจักรและค่า Indicator ปัจจุบัน
*   **Section C**: ประวัติ ISO Zone แต่ละรอบ
*   **Section D**: Top Frequencies รอบล่าสุด
*   **Section E — Recommendations**: สรุปคำแนะนำ เช่น ควรหยุดเครื่องซ่อมทันที (URGENT) หรือเฝ้าระวัง (CAUTION) รวมถึงรอบการตรวจสอบ (เช่น รายสัปดาห์, รายเดือน)

## 5. OUTLIER REMOVAL DETAIL
รายละเอียดเชิงลึกเกี่ยวกับขั้นตอนการตัดสัญญาณรบกวน (Data Cleaning)
*   **IQR k, Q1, Q3, Fence Low/High**: พารามิเตอร์เชิงสถิติที่ใช้กำหนดกรอบว่าข้อมูลไหนถือเป็น Spike (เกณฑ์ IQR)
*   **Peak Raw / Peak After**: ค่า Peak ก่อนและหลังตัด Spike

## 6. ANOMALY DETECTION (Isolation Forest)
การใช้ Machine Learning (Isolation Forest) ในการตรวจจับความผิดปกติแบบไม่มี Label
*   **Anomaly Score**: คะแนนความผิดปกติ (ยิ่งติดลบยิ่งถือว่ามีพฤติกรรมแปลกไปจากรอบเวลาอื่นๆ)
*   **Status**: 🔴 ANOMALY หรือ ✅ Normal

## 7. CF TREND FORECAST
การพยากรณ์ล่วงหน้าด้วย Linear / Polynomial Regression เพื่อหาวันที่คาดว่าความรุนแรง (Crest Factor) จะถึงจุดอันตราย
*   **Type**: Actual (ข้อมูลจริง) / Forecast (ข้อมูลพยากรณ์อนาคต)
*   **CF Linear Fit / CF Poly Fit**: ค่าที่เส้นพยากรณ์วาดได้
*   **Zone (Linear)**: โซนความอันตราย (Zone D คือเลยขีดสุด) ในเดือนถัดๆ ไป

## 8. ZONE CLASSIFICATION (Random Forest)
จำลองเปรียบเทียบการจัดโซนอันตราย (Zone) ระหว่างระบบ Rule-based (เขียนสูตรแบบ IF-ELSE ของ ISO) และโมเดล Machine Learning (Random Forest)
*   **Rule Zone / RF Zone**: โซนจากวิธีตั้งกฎ vs โซนจากโมเดล ML
*   **Match**: ตรงกันหรือไม่ (Yes/No)
*   **P(Zone A/B/C/D)**: ความน่าจะเป็นของแต่ละโซนที่ AI ทำนายออกมา

### วิธีการเรียกใช้งาน (รันโค้ด)
1. นำไฟล์ข้อมูล Waveform นามสกุล `.txt` (เช่น Jun24, Sep24, Oct24) ไปวางรวมกันในโฟลเดอร์เดียวกับโค้ด `main.py`
2. ติดตั้งไลบรารีที่จำเป็น (ถ้ายังไม่อัปเดต) ด้วยคำสั่ง `pip install -r requirements.txt`
3. รันโปรแกรมด้วยคำสั่ง `python main.py`
4. เมื่อโปรแกรมรัน จะมีหน้าต่าง (Popup) เด้งขึ้นมา ให้ลากคลุมเลือกไฟล์ `.txt` ทั้ง 3 รอบพร้อมกัน แล้วกด Open
5. รอโปรแกรมประมวลผลสักครู่ เมื่อเสร็จสิ้นจะเซฟทั้งไฟล์รูปกราฟ `.png` และไฟล์ `.csv` (ไฟล์นี้) ลงในโฟลเดอร์ `output/`
